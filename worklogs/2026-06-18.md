## 判断

- Ingress Controller は導入せず、Cilium Gateway API を採用する
- Gateway API CRD は Cilium のサポート matrix に合わせ、Kubernetes Gateway API の最新版ではなく Cilium 1.19 互換の v1.4.1 に pin する
- TLSRoute は Gateway API の experimental channel にあるため、standard CRD と取得元を分けて管理する
- BackendTLSPolicy は Cilium 1.19 Gateway API の必要 CRD ではないため、現時点では管理対象に含めない
- Gateway は platform の責務、HTTPRoute は apps の責務
- TLS termination 用の wildcard Certificate は platform の責務として Gateway と同じ namespace に置く
- Gateway 用 TLS は staging issuer ではなく production issuer で管理する
- L2 Announcement は public VLAN（eth1）、gateway worker（wk-01, wk-02）のみで行う
- control-plane は public ingress path から外す
- 163.220.236.73 を shared HTTPS Gateway に使う
- .74-.76 は将来の Gateway / L4 Service 用に予約
