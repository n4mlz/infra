# 2026-06-02

## 実施内容

- Flux を Dockerfile に追加し、devcontainer を再構築した
- `kubernetes/flux/` を Flux root として bootstrap した（`--token-auth` で最小権限）
- `kubernetes/platform/` にクラスタ基盤を構成した
- `kubernetes/apps/` にアプリケーション workload を構成した
- Cilium を Flux HelmRelease 管理へ移行した（KubePrism `localhost:7445`, Cilium `1.19.4`）
- smoke-test app を Git 管理下に置いた
- GitHub Flux Bootstrap Token を 1Password に追加し、credential-handling ドキュメントを更新した
- 構成の再編成（`clusters` → `flux`, `infrastructure` → `platform`）を実施し、Kustomize 参照原則を確立した
- runbook `3-flux-gitops-bootstrap.md` を作成した

## 判断

- クラスタは dev/prod の区別がないため単一の `kubernetes/` 配下に `flux`, `platform`, `apps` の 3 層構成
- Cilium values は HelmRelease に inline し、`talos/cilium-values.yaml` は初期 Talos bootstrap 時の一時ファイルとして残す
- Flux bootstrap は `--token-auth` を使用し、Administration read-only で済む構成にした

## Kustomize 参照原則

- 親は子だけを見る
- 子は親や兄弟を見ない
- 横参照しない
- Flux が管理するパスの外を参照しない
